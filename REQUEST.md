# REQUEST

## Story

- AWS上の1つのアカウントに**複数プロジェクト**が存在し、**同じロググループ**にログを出力している
- ログストリーム名にプロジェクト識別子が含まれるが、ログ本文には含まれない
- 各プロジェクトごとに**特定のキーワードを監視**し、SNSｰ>Chatbotｰ>Slackに通知したい
- プロジェクトごとにキーワード、緊急度、除外リスト、通知先、検索期間、通知間隔などカスタマイズしたい
- 将来的にはロググループをプロジェクトごとに分離する予定

## Design

- 監視
    - 複数プロジェクトのキーワード検知を１つのLambdaで実現
    - LambdaでBoto3のAPIを使用、柔軟に検索したいため（除外やソートなど）、logs insightsのqueryで検索
    - コスト面を考慮し、必要最小限なスキャンデータでqueryを実行
    - LambdaはDynamoDBから設定を読み取り、プロジェクト単位で検索する
    - 検索は一定間隔で行い、EventBridgeのschedulerで間隔を設定する
    - 検索結果はChatbotのカスタムメッセージ専用スキーマに沿って整形する
    
- 設定
    プロジェクト単位で設定できる項目
    - ロググループ名（パターン）
    - ログストリーム名（パターン）
    - 緊急度（INFO, WARNING, ERROR, CRITICAL)
    - キーワード（同一緊急度のキーワードはリストに入れる）
    - 除外リスト（検索結果の中に通知不要のものを除く）
    - 定期実行間隔
    - 通知間隔（silence period、同じのキーワードが通知されてからどれくらいの時間に重複通知しない、除外リストのヒットはカウントしない）
    - 検索期間（time range,queryはどれくらいの時間帯を検索する）
    - 通知先（緊急度によって別々の通知先に設定できる）
    - 通知文（緊急度・キーワードごとにカスタマイズできるテンプレート）
    - 復旧時通知要否
    - 復旧時通知文（テンプレート）
    
    全体的に（GLOBAL）設定できる項目(いわゆるfall back, default)
    - 定期実行間隔(5分)
    - 通知間隔(5分)
    - 検索期間（過去５分）
    - 通知先
    - 通知文（テンプレート）
    - 復旧時通知要否（defaultは要)
    - 復旧時通知文（テンプレート）
    
    
- 通知
    - 検索結果の中に通知不要のものを除いて、残りのデータを整形する
    - 通知文は設定できる
    - 通知文テンプレートの例
===
[緊急度] {プロジェクト名} {キーワード}を検出
(オプション: 特定の緊急度はmention)
{ロググループ}
{ログストリーム}
{検出回数}
検知内容：
    ```
    {検知したログ}
    ```
コンテキスト：
    ```
    {検知したログ直前のログ}(行数も設定可能)
    ```
===
    - 同じキーワードが複数検出した場合、重複を除いて最初のログ内容を送信（検出回数を記録）
    - 通知はSNSを入口とする
    - SNS -> Chatbot -> Slack
    - SNS -> Email　　
    
- 状態管理
    - DynamoDBで設定を管理する
    - GLOBAL設定とPROJECT設定を分ける
    - 1プロジェクトは1レコードで、JSON形式の設定ファイルを持つ
    - 状態（前回検索した時間、通知したかどうかなど）もDynamoDBで持つが、手動設定ではなく、自動更新
